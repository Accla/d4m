Constraints

Cloudbase supports constraints. Constraints are applied to mutations at ingest time.

Implementing a new constraint is a snap. Simply write some Java code that implements cloudbase.core.constraints.Constraint.

To deploy a new aggregator, jar it up and put the jar in cloudbase/lib/ext.

After creating a constraint, set a table specific property to deploy it. The following example adds two constraints to table foo. In the example com.test.ExampleConstraint and com.test.AnotherConstraint are class names. 

> config -t foo cloudbase.table.constraint.1=com.test.ExampleConstraint 
> config -t foo cloudbase.table.constraint.2=com.test.AnotherConstraint
> config -t foo cloudbase.table.constraint
TYPE       | NAME                                                         | VALUE
-----------+--------------------------------------------------------------+---------------------
table      | cloudbase.table.constraint.1                                 | com.test.ExampleConstraint
table      | cloudbase.table.constraint.2                                 | com.test.AnotherConstraint
